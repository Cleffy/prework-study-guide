Prework Study Guide

HTML

  • HTML is the frame of a webpage.
  • It's preferable to use semantic elements.
  • The head element contains information about the webpage.
  • The meta elements help search engines and browsers process the website.
  • The title element is displayed on the browser tab.
  • The body element represents the visible content shown to the user.
  • The header element defines the beginning section of the website.
  • The main element is the website's main section.
  • The footer element is displayed at the bottom of the website.
  • The section element defines sections.
  • The ul and li elements are used for lists.
  • The image element adds images.
  • The p element adds blocks of text.
  • The br element creates a linebreak.
  • The a element creates hyperlinks.
  • The h element creates headers, they should be used in descending order.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • We should reduce redundant code by combining edits with a comma.
  • We can use classes to define certain elements.

Git

  • git status: checks what branch we are currently on.
  • git checkout -b branch-name: creates a new branch and switches to it.
  • git add -A: stages the changed files.
  • git commit -m: commits the changes to the repository.
  • git pull origin main: updates to the current branch.
  • git push origin: pushes changes to the repository.
